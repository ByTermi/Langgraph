Qué hace cada nodo

summary_subgraph

Genera/amplía un resumen para el próximo turno. Estado privado: clean_messages.

filter_messages
messages Elimina AIMessage con tool_calls y ToolMessage. Guarda resultado en clean_messages (privado).
↓ clean_messages → call_llm
call_llm
clean_messages summary Invoca LLM. Devuelve summary actualizado y añade a parallel_outputs.

analysis_subgraph

Extrae métricas y genera insight con LLM. Estado privado: metrics.

extract_metrics
messages Cuenta mensajes por rol y llamadas por tool. Guarda dict en metrics (privado).
↓ metrics → generate_insight
generate_insight
metrics LLM genera frase describiendo patrón de uso. Añade a parallel_outputs.

map-reduce (Send API)

Evalúa todas las tareas pendientes en paralelo. Estado privado por tarea: TaskAssessState.

list_pending_tasks
Lee DB: SELECT … WHERE completada=0. Guarda lista en pending_tasks.
↓ send_to_assess (edge fn) → [Send×N] o ["consolidate_tasks"]
assess_task ×N
TaskAssessState Cada Send lleva {task_id, task_desc}. LLM evalúa urgencia. Añade a task_assessments[ ] vía operator.add.
↓ todos los assess_task convergen aquí (fan-in automático)
consolidate_tasks
task_assessments LLM genera informe de prioridad ordenado. Escribe task_report al estado padre.

Cómo se declaran

Subgrafos: StateGraph con output_schema para filtrar claves privadas

class SummarySubgraphState(TypedDict):
    messages: List[BaseMessage]  # compartida
    summary: str                 # compartida
    clean_messages: List[...]    # PRIVADA — no en State padre

class SummarySubgraphOutputState(TypedDict):
    summary: str           # qué se escribe al padre
    parallel_outputs: List[str]

_b = StateGraph(SummarySubgraphState,
              output_schema=SummarySubgraphOutputState)
_b.add_node("filter_messages", filter_messages_node)
_b.add_node("call_llm", call_llm_node)
_b.add_edge(START, "filter_messages")
_b.add_edge("filter_messages", "call_llm")
_b.add_edge("call_llm", END)
summary_subgraph = _b.compile()

Map-Reduce: Send API con estado privado por invocación

class TaskAssessState(TypedDict):  # estado privado del Send
    task_id: int
    task_desc: str

def send_to_assess(state):  # edge function — MAP
    tasks = state.get("pending_tasks") or []
    if not tasks:
        return ["consolidate_tasks"]  # skip map
    return [
        Send("assess_task", {"task_id": t["id"], "task_desc": t["desc"]})
        for t in tasks     # 1 Send por tarea → N paralelos
    ]

# Nodo MAP: recibe TaskAssessState vía Send
def assess_task(state: TaskAssessState):
    response = llm.invoke(...)
    return {"task_assessments": [f"[ID {state['task_id']}] ..."]}
    # → operator.add acumula en el padre

builder.add_conditional_edges(
    "list_pending_tasks", send_to_assess,
    ["assess_task", "consolidate_tasks"],
)
builder.add_edge("assess_task", "consolidate_tasks")
builder.add_edge("consolidate_tasks", END)

Estado padre con todas las claves compartidas y reducers

class State(MessagesState):
    summary: str                                                  # ← summary_subgraph escribe
    parallel_outputs: Annotated[List[str], operator.add]          # ← summary + analysis acumulan
    pending_tasks: List[dict]                                     # ← list_pending_tasks escribe
    task_assessments: Annotated[List[str], operator.add]          # ← assess_task×N acumulan (MAP)
    task_report: str                                              # ← consolidate_tasks escribe (REDUCE)

def route_from_agente(state) -> list[str]:
    if _has_pending_tool_calls(state["messages"]):
        return ["tools"]
    return ["summary_subgraph", "analysis_subgraph", "list_pending_tasks"]  # 3 ramas

Reglas clave

Claves solapadas

LangGraph sincroniza padre↔subgrafo por nombre de clave. messages y summary en SummarySubgraphState coinciden con State → comunicación automática.

output_schema = filtro

Solo las claves del output_schema se escriben al padre. clean_messages y metrics no están en ningún output_schema → nunca salen del subgrafo.

Send API (map)

Send("nodo", estado) lanza una invocación independiente con su propio estado privado (TaskAssessState). El número de invocaciones es dinámico: se decide en tiempo de ejecución, no de compilación.

operator.add (reduce)

Cada assess_task devuelve {"task_assessments": [...]}. Como el padre tiene Annotated[List, operator.add], LangGraph concatena todos los resultados paralelos automáticamente antes de llegar a consolidate_tasks.
